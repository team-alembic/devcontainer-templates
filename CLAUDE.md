# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

Dev Container Templates for Elixir development, published to GHCR at `ghcr.io/team-alembic/devcontainer-templates/`. Two templates:

- **elixir-asdf** — standalone Elixir devcontainer using asdf for version management
- **elixir-asdf-postgres** — same, plus a PostgreSQL service container with docker-compose

## Testing

Smoke tests build a devcontainer and run assertions inside it. Requires Docker and `@devcontainers/cli`.

```bash
# Build and test a template locally (two separate steps)
./.github/actions/smoke-test/build.sh elixir-asdf
./.github/actions/smoke-test/test.sh elixir-asdf

# Same for the postgres template
./.github/actions/smoke-test/build.sh elixir-asdf-postgres
./.github/actions/smoke-test/test.sh elixir-asdf-postgres
```

`build.sh` substitutes `${templateOption:*}` placeholders with defaults from `devcontainer-template.json`, copies test fixtures into the container, then runs `devcontainer up`. `test.sh` execs the template's `test/<template>/test.sh` inside the running container.

Per-template test assertions live in `test/<template>/test.sh` and use `test/test-utils/test-utils.sh` for the `check`/`reportResults` helpers.

## Publishing

Manual only — run the **Release Dev Container Templates** workflow from the GitHub Actions tab. It publishes to GHCR and auto-generates `README.md` files in each `src/*/` directory via a PR.

## Architecture

```
src/<template>/
  devcontainer-template.json   # Template metadata, options, version
  .devcontainer/
    devcontainer.json           # Devcontainer config with ${templateOption:*} placeholders
    setup.sh                    # onCreateCommand — installs asdf plugins, tools, hex, rebar
    docker-compose.yml          # (postgres template only)
  NOTES.md                      # Internal notes
  README.md                     # Auto-generated by release workflow — do not edit manually

test/<template>/
  .tool-versions                # Fixture for smoke tests
  test.sh                       # Assertions run inside the built container
```

Template options use `${templateOption:optionName}` placeholders in devcontainer files. `build.sh` replaces these with defaults from `devcontainer-template.json` during testing.

Both templates share the same base image (`mcr.microsoft.com/devcontainers/base:debian`) and devcontainer features (asdf, node, github-cli, claude-code, apt build deps). The postgres variant adds docker-compose with a `db` service and a `dialyzer-plt` volume.

## CI

- **test-pr.yaml** — runs on push to `main` and on PRs. Uses `dorny/paths-filter` to detect which templates changed and only smoke-tests those.
- **release.yaml** — manual dispatch only, from `main`. Publishes templates to GHCR using `devcontainers/action@v1` and creates a PR for generated docs.

## Key Conventions

- `setup.sh` files use `${templateOption:*}` syntax — these are replaced at template-apply time, not at build time
- The `README.md` inside each `src/*/` is auto-generated — edit `NOTES.md` or the root `README.md` instead
- PostgreSQL credentials are always `postgres`/`postgres` with database `<projectName>_dev`
- `DATABASE_HOST=db` is set in both `docker-compose.yml` and `devcontainer.json` remoteEnv
